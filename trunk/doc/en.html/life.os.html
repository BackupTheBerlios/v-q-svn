3.3. Operating system settings

This section describes tweaking operating system of a client and a server. It also describes problems related with some operating systems and configurations.

3.3.1. OpenBSD

3.3.1.1. Setting open files limit

During tests it was noticed that qmailvqd sometimes hangs and starts eating all CPU time. It may happen even under low load. It seems that it is a problem related with open files limit set to low.

Note: This problem occurs only when using MICO.

If you encounter this problem you should set larger number of files that can be opened by qmailvqd, you can do that by creating special users' class in /etc/login.conf: 

vq:\
 :openfiles-cur=768:\
 :tc=daemon:

To refresh binary version of this file (/etc/login.conf.db) execute following command: 

cap_mkdb /etc/login.conf

Remeber to change class to "vq" for user running qmailvqd.
